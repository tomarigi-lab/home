# Project Guidelines

## Claude Codeの役割

- `handoff/requests/` に置かれた依頼を読み、指定範囲内で調査・制作・実装・テストを行う。
- 実行結果は `handoff/results/` にMarkdownで残す。
- 未決事項を勝手に確定せず、判断が必要な点を結果へ残す。

## プロジェクト境界

- 原則として、この `home` プロジェクト内だけを読み書きする。
- 親フォルダ、兄弟プロジェクト、`codex-lab` を勝手に参照・編集しない。
- 外部ファイルが必要な場合は、対象と目的を結果へ書き、人間の確認を待つ。

## GitHub Pages公開範囲

- GitHub Pagesの公開元は、`main`ブランチの`/docs`フォルダとする。
- Webサイトとして公開するHTML、CSS、画像などは `docs/` 内に置く。
- `CLAUDE.md`、`project/`、`handoff/` はWebサイトとして配信しない。
- リポジトリを公開する場合、Pagesの対象外でもGitHub上では閲覧できるため、秘密情報を置かない。

## 実行ルール

- 依頼書に書かれた「今回やること」「編集可能な範囲」「禁止事項」を優先する。
- 依頼書にない大規模変更を行わない。
- 作業前に、対象ファイルと完了条件を確認する。
- 作業後に、変更ファイル、確認結果、未決事項、次の判断を結果へ残す。
- 作業後に `handoff/project-status.md` を更新し、次回作業の開始地点を明確にする。

## ローカル表示確認

- 静的ホームページの表示確認には、Python標準の簡易Webサーバーを使う。
- プロジェクトルートで `python3 -m http.server 8000 --directory docs` を実行する。
- 確認URLは `http://localhost:8000`。
- 確認後はサーバーを停止する。
- 公開環境ではなく、MacBook内での表示確認用途として扱う。

## 人間確認が必要な操作

- Git push、GitHub Pages公開、本番反映。
- ファイル削除、履歴書き換え。
- 外部サービスへの送信・公開。
- 購入、契約、課金。
- 秘密情報、個人情報、会社情報の利用。
- プロジェクト外へのアクセス。

## ルーティンと非定型タスク

- 初めて行う作業、通常と異なる作業、影響範囲が不明な作業は、実行前に計画と影響を結果へ残し、人間確認を待つ。
- 同じ作業を複数回安全に実施し、人間が承認済みルーティンとして定義した場合のみ、確認を減らしてよい。
- 以前実施した作業でも、対象・範囲・目的が変わる場合は非定型タスクとして扱う。
- Git push、GitHub Pages公開、本番反映、削除、履歴書き換えは、ルーティン化後も依頼書の明示的な許可がない限り実行しない。

## Handoff

```text
handoff/
├─ project-status.md  現在地と次の作業
├─ requests/          Codexが作成した依頼
└─ results/           Claude Codeが作成する実行結果
```

結果ファイルには最低限、次を含める。

- 実施したこと。
- 変更したファイル。
- 確認・テスト結果。
- 未実施・未決事項。
- リスク・注意点。
- 次に人間が判断すること。
